審稿會議

2 月 7 日晚上，審稿委員們依然挑燈夜戰，在眾多優秀的稿件中忍痛取捨著。

議程組長 Pellaeon 表示：「今年的稿件不僅優秀，數量也相當的多，讓我們非常苦惱議程的安排，不過這也代表，SITCON 已經受到越來越多人的關注。」審稿會議是由評分委員事先看過所有投稿議程之後，在會議中針對分數較高的一一討論，並安排進不同的演講廳。評分標準則是以稿件的內容、酷炫程度以及表達能力等三個項目為主，其次會考慮稿件主題的重複性，因為我們由衷地希望會眾能夠接收到不同主題的演講。最大的國際會議廳通常會放委員們覺得最能吸引大家的議程。會後也將有問卷調查詢問會眾對議程的評價，進而讓往後的評委有所依據，到時還請大家一定要填寫哦！

議程內容

PN Wu (小平) - 為學校網站系統做測試甘苦談

學校網站有一大堆系統，選課、教學、註冊系統劈哩啪啦的一大堆，這些往往都是各處室各管各的，連計中主任都莫可奈何，有時候還會把一堆未經測試的修改直接上線，臭蟲讓使用者用到傻眼。小平將帶大家看一些慘痛的經驗，告訴大家這些問題的原因和解決方案，以及在 VS、PHP、Java 開發測試時的具體方法。

AJ - 自己的異質多核心平台自己幹

異質多核心平台是藉由不同構造設計的處理器，分開處理系統與應用程式和資料，榨乾硬體資源來達到最高的效率。現在在伺服器、遊戲、移動裝置等產業上已經大量運用這樣的技術，將 GPU 拿來做大量平行運算。其實這項技術並非遙不可及，就讓 AJ 來告訴大家如何只用自家電腦裡的 GPU 實作這樣的技術。

和風信使 - 在 DigitalOcean 架設 GitLab

和其他人合作要寫大型專案時，難保會不知不覺中讓小 Bug 鑽進 Code 裡，為了能加速和精確 Debug，版本控制是非常重要的。說到版本控制，很多人應該都會直覺想到 GitHub 吧！可是 GitHub 使用上有一些功能要收費，不然就是限制私人 Project 數量，另一大服務商 BitBucket 也是限制免費用戶的共同開發帳號數量，既然有這樣的需求，何不自己用更低廉的價格，在租來的雲端主機上也架一個自由的 GitLab 出來呢？

民 2 - From Android NDK To AOSP

NDK (Native Development Kit)是一個原生開發工具包，它可以讓你在許多應用程序上使用 C 或 C++ 等語言在許多應用程序上，然而，NDK 是無法對大部分 App 進行改善的；AOSP (Android Open Source Project) 則是 Android 作業系統的開放原始碼專案所成立的一個計劃，讓全球各地的開源社群來負責開發 Android 應用程式和作業系統來延長和擴展 Android 的功能和效能；現在就讓我們來看看 民2 是怎麼從 NDK 到 AOSP 的過程吧！

shunyi - Android 重製 App 偵測系統

現在人手一支智慧型手機，Android 的市佔率將近七成，每天上架的 App 更是超過七百個，用戶在使用行動裝置時，往往看到喜歡的 App 就立刻下載；但是你知道嗎？其實手機跟電腦一樣，都是會中毒的，更不用說這些令人眼花撩亂的 App 中有許多是惡意程式，它可能使你的手機資料外流而不自知，甚至會讓電話費暴增，這些後果都不是任何一個用戶所樂見的。就讓我們來看 shunyi 如何用Android 重製 App 偵測系統，將惡意程式辨別出來，以免用戶的權益受損的情況發生。

聽風 - 第一次查課程評價就上手

身為大學生，多少都會經歷「選課大戰」的洗禮。除了搶課之外，避免淪為毒師刀下的亡魂，或選到與自己興趣或教學風格不合的課程，也是選課大戰的一環。人人無所不用其極的打探各門課程的評價與心得，更是大學生都曾經有過的共同經驗。然而，散亂在 Facebook、PTT 等平台上的資料，除了逐一翻找，是否能夠有更高效率的方法？讓聽風來教大家，怎麼以更方便的管道取得並彙整各種平台上的評價精髓吧！
